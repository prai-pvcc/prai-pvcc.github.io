  • Von Neumann Computer architecture, 3 characteristic:
    1. Four major subsystems
    2. Stored program concepts
    3. Sequential execution of instructions
  1. Computer Architecture Terms
    1. Ram
      2. RAM is the main memory in a computer.
    2. Cache
      3. cache is a high-speed data storage layer which stores a subset of data
    3. Principle of locality: The high likelihood that when a program fetches a piece of data or an insttruction:
      1. one
      2. two
    4. ROM
      5. computer storage containing non-volatile, permanent data that, normally, can only be read, not written to. ROM contains the programming that allows a computer to start up or regenerate each time it is turned on.
    5. Memory cell size
      6. Memory cells can be of different sizes: 8 bits, 16 bits, 32 bits, 64 bits. Different sized cells are used to store different types of data. A cell of 8 bits is called a byte. An integer is a whole number. An integer is generally stored in 4 bytes, or 32 bits.
    6. memory cell address
      7. Memories consist of a number of cells (or locations) each of which can store a piece of information. Each cell has a number, called its address, by which programs can refer to it.
    7. Maximum memory size
      8. The max memory size is the maximum total RAM that the processor will address.
    8. Register
      9. one of a small set of data holding places that are part of the computer processor. A register may hold an instruction, a storage address, or any kind of data (such as a bit sequence or individual characters).
    9. Fetch
      10. the retrieval of data by a software program, script, or hardware device. After being retrieved, the data is moved to an alternate location or displayed on a screen.
    10. Store
      11. saves the results to a Register or Memory.
    11. Memory access time
      12. how long it takes for a character in RAM to be transferred to or from the CPU. Fast RAM chips have an access time of 10 nanoseconds (ns) or less.
    12. DASD
      13. A direct-access storage device (DASD) is another name for secondary storage devices that store data in discrete locations with a unique address, such as hard disk drives, optical drives and most magnetic storage devices.
    13. Arithmetic and logic unit(ALU)
      14. the part of a central processing unit that carries out arithmetic and logic operations on the operands in computer instruction words.
    14. Control Unit
      15. circuitry within a computer's processor that directs operations. It instructs the memory, logic unit, and both output and input devices of the computer on how to respond to the program's instructions.

Chapter 6:

  • Four things the interface between the computer and the hardware does:
    1. Provide a shared boundary for exchanging information between software and hardware components.
    2. Act as an intermediary between the computer user and the hardware by controlling the execution of programs.
    3. Control the hardware through an operating system, which includes drivers for various hardware components, such as memory, CPU, and fans/cooling.
    4. Allow for the erasure of data and programs from the hardware through specific commands or tools provided by the operating system.
  • Responsibilities of systems software:
    1. Analyzing functional and systems requirements: Systems software developers need to analyze the needs of the business or organization and determine the requirements for the software system.
    2. Developing application and database architecture and design: Once the requirements are understood, systems software developers need to design the architecture and framework of the software system.
    3. Writing code: Systems software developers write the code for the software system using programming languages such as C++, Java, Python, or Ruby. They need to ensure that the code is efficient, scalable, and secure.
    4. Validating data: Systems software developers need to validate the data that is input into the software system. This involves checking for errors and inconsistencies, ensuring that the data meets the required standards, and verifying that the data is accurate and complete.

Terms

  1. Systems software
    2. a program designed to run a computer's hardware and applications and manage its resources, such as its memory, processors, and devices.
  2. User interface
    3. the space where humans and machines interact with each other. It serves as the point of communication and interaction between the user and the device, website, or application.
  3. Language services
    4. range of services that enable effective communication between people who speak different languages. These services are typically provided by bilingual personnel or language assistance vendors, and they may include interpretation services and/or translation services.
  4. Memory managers
    5. software utility that works with the operating system to help manage computer memory more efficiently.
  5. Information managers
    6. the process of collecting, storing, managing, maintaining, and distributing data and other types of information from one or more sources. It involves the complete lifecycle of information, from identification and collection to storage, retrieval, and dissemination.
  6. I/o managers
    7. It serves as a bridge between various applications, system components, and different types of devices, such as virtual, logical, and physical devices.
  7. Scheduler
    8. a scheduler is a software tool that assists in the scheduling and tracking of computer batch tasks. These tasks can include running specific programs, backups, and system maintenance.
  8. Utilities
    9. software that performs specific tasks related to the management of computer functions, resources, or files. These tasks may include virus protection, password management, memory management, and file management.
  9. Assembly langauge
    10. Assembly language programming is a type of low-level computer programming language that enables programmers to communicate directly with a computer's hardware.
